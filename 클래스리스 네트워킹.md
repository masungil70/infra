  1. 클래스리스 네트워킹(Classless Networking)이란?

  클래스리스 네트워킹은 IP 주소를 할당하고 라우팅(경로 설정)하는 현대적인 방식으로, 기존의 고정된 '클래스(Class)' 개념을 없애고
  네트워크의 크기를 필요한 만큼 자유롭게 조절할 수 있게 만든 유연한 주소 체계입니다.

  핵심 기술은 CIDR(사이더, Classless Inter-Domain Routing)이며, 클래스리스와 CIDR은 거의 같은 의미로 사용됩니다.                
  
  2. 왜 등장했는가? (클래스풀 방식의 문제점)

  클래스리스를 이해하려면 그 이전에 사용되던 클래스풀(Classful) 방식의 문제점을 알아야 합니다.

  과거에는 IP 주소를 A, B, C, D, E 라는 5개의 클래스로 나누어 관리했습니다. 이 중 A, B, C 클래스가 주로 사용되었는데, 각 클래스는
  네트워크의 크기가 고정되어 있었습니다.

   * A 클래스: 1개의 네트워크에 약 1,600만 개의 IP를 가짐 (대규모)
   * B 클래스: 1개의 네트워크에 약 65,000개의 IP를 가짐 (중규모)
   * C 클래스: 1개의 네트워크에 254개의 IP를 가짐 (소규모)

  문제점: 극심한 IP 주소 낭비

  예를 들어, 어떤 회사가 500개의 IP 주소가 필요하다고 가정해 봅시다.
   * C 클래스(254개)는 너무 작아서 쓸 수 없습니다.
   * 어쩔 수 없이 B 클래스(65,000개)를 할당받아야 합니다.
   * 결과: 500개만 사용하고, 나머지 약 64,500개의 IP 주소는 그대로 낭비되었습니다.                                                
                                                                                                                             
  이러한 비효율성 때문에 IPv4 주소가 예상보다 훨씬 빠르게 고갈될 위기에 처했고, 이를 해결하기 위해 클래스리스 방식이 등장했습니다.

 ---

  3. 어떻게 동작하는가? (핵심 개념)

  클래스리스 네트워킹은 두 가지 핵심 기술을 사용합니다.

  가. VLSM (Variable Length Subnet Masking, 가변 길이 서브넷 마스크)

  클래스풀 방식에서는 서브넷 마스크가 /8, /16, /24로 고정되었지만, VLSM은 서브넷 마스크의 길이를 1비트 단위로 자유롭게 조절할 수
  있게 합니다. 이를 통해 네트워크를 필요한 크기에 맞게 '잘게 쪼개' 쓸 수 있습니다.

   * 예시:
       * 500개의 IP가 필요하다면? -> /23 (510개) 네트워크를 할당 (낭비 최소화)
       * 100개의 IP가 필요하다면? -> /25 (126개) 네트워크를 할당
       * 30개의 IP가 필요하다면? -> /27 (30개) 네트워크를 할당
       * 라우터 간 연결처럼 2개의 IP만 필요하다면? -> /30 (2개) 네트워크를 할당

  이처럼 낭비되는 IP 주소를 획기적으로 줄일 수 있습니다.                                                                           

  나. CIDR 표기법 (Prefix Notation)
  클래스리스 방식에서는 네트워크를 표현하기 위해 IP 주소 뒤에 /와 숫자를 붙이는 CIDR 표기법을 사용합니다.

  `192.168.10.0 /24`

   * /24는 IP 주소의 32비트 중 앞에서부터 24비트가 네트워크 주소 부분임을 의미합니다.
   * 나머지 8비트(32-24)는 호스트(PC, 서버 등) 주소 부분입니다.

  | CIDR 표기법|서브넷 마스크|할당 가능한 호스트 수|
  |---|---|---|
  | /24  |255.255.255.0    | 254개|
  | /25  | 255.255.255.128 | 126개|
  | /26  | 255.255.255.192 | 62개 |
  | /27  | 255.255.255.224 | 30개 |
  | /30  | 255.255.255.252 | 2개  |
  

  
  4. 또 다른 핵심 장점: 경로 요약 (Route Summarization)

  클래스리스 방식은 '슈퍼넷팅(Supernetting)'이라고도 불리는 경로 요약을 가능하게 합니다. 이는 인터넷의 효율성을 극대화하는 매우
  중요한 기능입니다.

   * 문제: 만약 인터넷의 모든 라우터가 전 세계의 수백만 개에 달하는 작은 C 클래스 네트워크 경로 정보를 모두 저장해야 한다면,
     라우터의 메모리와 CPU에 엄청난 부담이 가고 인터넷 속도가 느려질 것입니다.

   * 해결 (경로 요약): 예를 들어, 한 ISP(인터넷 서비스 제공자)가 203.0.113.0/24 부터 203.0.113.255/24 까지 256개의 /24 네트워크를
     관리한다고 가정합시다.
       * 클래스풀 방식: 256개의 경로 정보를 각각 인터넷에 광고해야 합니다.
       * 클래스리스 방식: 이 256개의 네트워크를 `203.0.113.0/16` 이라는 단 하나의 경로 정보로 요약하여 인터넷에 광고할 수 있습니다.

  전 세계의 라우터들은 단 하나의 요약된 경로 정보만 저장하면 되므로, 라우팅 테이블의 크기가 크게 줄어들고 인터넷 전체의 라우팅
  효율이 비약적으로 향상됩니다.

  5. 클래스풀 vs 클래스리스 비교 요약                                                                                         
 | 구분 | 클래스풀 (Classful)  | 클래스리스 (Classless) |
 |---|---|---|
 |네트워크 구분 | A, B, C 등 고정된 클래스  | 클래스 개념 없음                   |
 |서브넷 마스크 | /8, /16, /24 로 고정     | /1 ~ /32 까지 자유롭게 사용 (VLSM) |
 |IP 할당      | 낭비가 매우 심함          | 필요한 만큼 할당하여 효율적        |
 |라우팅       | 경로 요약(슈퍼넷팅) 불가   | 경로 요약(슈퍼넷팅) 가능          |
 |표기법       | 클래스로 구분             | CIDR 표기법 (/xx) 사용            |

  클래스리스 네트워킹(CIDR)은 낡고 비효율적인 클래스풀 시스템을 대체한 현대 인터넷의 근간입니다. VLSM을 통해 IP 주소 고갈 문제를
  완화하고, 경로 요약을 통해 인터넷 라우팅 시스템이 확장 가능하고 효율적으로 유지될 수 있도록 만든 핵심적인 기술이라고 할 수        
  있습니다.
