# 14. spine1 spine2, leaf1, leaf2, Ceph-C1,C2,C3,C4 서버 설정 검증

spine1 spine2, leaf1에 연결된 Ceph 서버 2대(C1,C2), leaf2을 연결된 Ceph 서버2대(C3,C4)를 사용하여 네트웍 문제를 검증합니다.

🔥 VLAN 10 / VLAN 20 / MTU 9000 / Spine-Leaf 구조가 정상인지 검증

## 🧭 전체 테스트 흐름

```
1. 물리 링크 확인
2. VLAN 인터페이스 구성
3. L2 통신 테스트 (ping)
4. MTU 테스트 (jumbo)
5. 대역폭 테스트 (iperf)
6. 장애 테스트 (failover)
```

---

# 1️. 물리 링크 확인

**Ceph Node3**에서 진행합니다


## ✔ 네트웍 설정 확인

```bash
ip a

```

- 설정된 관리용 ip 를 확인합니다
- 설정된 vlan 10의 ip를 확인합니다
- 설정된 vlan 20의 ip를 확인합니다

---

# 2️. L2 / VLAN 통신 테스트

## ✔ VLAN 10 테스트

Node3:


# 3️. MTU 9000 테스트

## ✔ jumbo frame 테스트

Node3:

```bash
ping -M do -s 8972 10.10.10.11
ping -M do -s 8972 10.10.10.12
ping -M do -s 8972 10.10.10.14
```

👉 정상:

```
no fragmentation
```

---

## ❗ 실패 시 의미

| 증상                 | 원인            |
| -------------------- | ----------      |
| packet fragmentation | MTU 불일치      |
| ping 실패            | 스위치 MTU 문제 |

---

# 4️. 대역폭 테스트 (iperf)

테스트 단계

1. Node1 (서버) -> Node4 (클라이언트) 대역폭을 확인 합니다
2. Node2 (서버) -> Node4 (클라이언트) 대역폭을 확인 합니다
3. Node3 (서버) -> Node4 (클라이언트) 대역폭을 확인 합니다

---

## 모든 노드에 iperf3 패키지를 설치

```bash
# iperf3 패키지를 설치
apt install -y iperf3

# iperf3 패키지를 서버로 실행하며 네트워크 속도를 Gbps 로 출력
iperf3 -s -f g

```

---

## ✔ Node4 (클라이언트)


# 5️. VLAN 분리 검증 (중요)

👉 일부러 틀리게 테스트

```bash
# Node1 (10.10.10.11) ↔ Node2 (10.10.10.12)
# Node1에서 실행 
# (정상)
ping 10.10.10.12  
```

```bash
# Node1 (10.10.10.11) ↔ Node2 (10.10.10.12)
# Node2에서 실행 
# (정상)
ping 10.10.10.11  
```

## 인터페이스 지정 ping

```bash
# Node1 (10.10.10.11) ↔ Node2 (10.10.10.12)
# Node1에서 실행 
# (정상)
ping -I vmbr1.10 10.10.10.12
```

-I 나간는 소스 인터페이스지정 옵션

## 실패

```bash
# Node1 (10.10.10.11) ↔ Node2 (10.10.10.12)
# Node2에서 실행 

# (실패)
ping -I vmbr1 10.10.10.12

# (실패)
ping -I vmbr1.30 10.10.10.12

```

# 🎯 문제 유형별 빠른 판단

| 증상         | 원인            |
| -------      | -------------   |
| ping 안됨    | VLAN mismatch   |
| MTU 실패     | 스위치 MTU      |
| 속도 낮음    | LACP/STP        |
| 간헐적 끊김  | loop / STP 문제 |
