### terraform 로그 출력 

terraform apply 실행 로그를 보는 방법은 다음과 같습니다.

1. 기본 출력 확인:

terraform apply 명령을 실행하면 모든 작업 내용(리소스 생성/변경/삭제 계획, 오류 메시지 등)이 기본적으로 터미널(표준 출력 및 표준 오류)에 표시됩니다. 가장 기본적인 로그 확인 방법입니다.

2. 상세 로그 활성화 (TF_LOG 환경 변수):

Terraform은 TF_LOG 환경 변수를 사용하여 더 자세한 로그를 생성할 수 있습니다. 이는 문제 해결 시 매우 유용합니다.

* 로그 레벨:
  * TRACE: 가장 상세한 로그 (디버깅에 가장 유용)
  * DEBUG: 디버그 정보 포함
  * INFO: 일반 정보
  * WARN: 경고 메시지
  * ERROR: 오류 메시지

  * 로그 파일을 저장:
    TF_LOG_PATH 환경 변수를 설정하여 로그를 파일로 저장할 수 있습니다. 이렇게 하면 터미널에 너무 많은 내용이 출력되는 것을 방지하고, 로그 내용을 나중에 분석하기 용이합니다.

Windows PowerShell에서 사용 예시:

```sh
# 로그 레벨을 TRACE로 설정하고, 모든 로그를 terraform.log 파일에 저장합니다.
$env:TF_LOG="TRACE"
$env:TF_LOG_PATH="terraform.log"

# 이 상태에서 terraform apply를 실행합니다.
terraform apply

# 로그 확인 (선택 사항: 파일 내용을 터미널에 출력)
Get-Content terraform.log | Select-Object -Last 50 # 마지막 50줄

# 또는
notepad terraform.log # 메모장으로 파일 열기

# 로그 환경 변수 해제 (선택 사항)
Remove-Item Env:TF_LOG
Remove-Item Env:TF_LOG_PATH
```

terraform apply 실행 전에 위 환경 변수를 설정하고 실행하면, terraform.log 파일에 매우 상세한 내부 동작 로그가 기록되어 문제의 원인을 파악하는 데 큰 도움이 될 것입니다.
