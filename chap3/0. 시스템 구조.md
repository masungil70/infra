## Proxmox 네트워크 준비 (매우 중요)

Proxmox NIC을 TRUNK 가능 상태로 변경
/etc/network/interfaces

```bash
auto lo
iface lo inet loopback

auto nic0 
iface nic0 inet manual

auto vmbr0
iface vmbr0 inet static
        address 192.168.3.2/24
        gateway 192.168.3.1
        bridge-ports nic0
        bridge-stp off
        bridge-fd 0
        bridge-vlan-aware yes  #Proxmox NIC을 TRUNK 가능 상태로 변경

source /etc/network/interfaces.d/*

----------------------------------

Proxmox 서버 물리적 및 논리적 배치 방법 

이미 Proxmox #1에 pfSense(라우터 VM)가 올라가 있는 상태이므로, 나머지 3대의 Proxmox 호스트(#2, #3, #4)는 네트워크적으로는 Proxmox #1과 수평적인 위치에 배치하되, 논리적으로는 pfSense의 하단에 위치하게 설정해야 합니다.
가장 권장되는 배치 및 설정 방법은 다음과 같습니다.

## 1. 물리적 배치 및 연결 (Hardware Layer)
나머지 3대의 서버도 관리형 스위치에 각각 연결합니다.

* 연결 방식: Proxmox #2, #3, #4의 물리 NIC를 스위치의 Trunk 포트에 연결합니다.
* 스위치 설정: 각 포트는 VLAN 10, 20, 30, 40 트래픽이 모두 통과할 수 있도록 Tagged(Trunk) 모드로 설정해야 합니다. 이렇게 해야 어떤 서버에서 VM을 생성하더라도 pfSense가 설정한 4개의 망에 즉시 접속할 수 있습니다.

## 2. 네트워크 논리 구조 (Logical Layer)
pfSense는 Proxmox #1 위에서 작동하지만, 스위치를 통해 모든 호스트의 게이트웨이 역할을 하게 됩니다.

* Traffic Flow:
* Proxmox #2의 VM → 스위치(Trunk) → Proxmox #1의 물리 포트 → pfSense(VLAN 라우팅/방화벽) → 인터넷(ER605)
* 이 구조 덕분에 Proxmox #1이 꺼지면 전체 네트워크가 중단될 수 있습니다. 이를 방지하려면 나중에 pfSense 고가용성(HA) 설정을 고려해야 합니다.

## 3. 나머지 3대에 꼭 해야 할 설정 (Configuration)
나머지 호스트들을 클러스터로 묶기 전에 아래 설정을 마쳐야 합니다.

   1. VLAN Aware Bridge 설정 (중요):
   * Proxmox #1에서 했던 것과 동일하게 각 서버의 vmbr0 설정에서 VLAN Aware 옵션을 켭니다. 그래야 각 호스트에 배정될 VM들이 VLAN Tag(10~40)를 달고 pfSense와 통신할 수 있습니다.
   2. 관리 IP 대역 통일 (VLAN 40):
   * 각 서버의 관리 IP(Proxmox GUI 접속용)를 VLAN 40(관리망) 대역으로 설정합니다.
      * 예: PVE #1(192.168.3.2), PVE #2(192.168.3.3), PVE #3(192.168.3.4), PVE #4(192.168.3.5)
   3. 클러스터 조인(Join):
   * PVE #1에서 클러스터를 생성한 후, 나머지 3대에서 PVE #1의 관리 IP(VLAN 40 대역)를 통해 Join Cluster를 수행합니다.
   
## 4. 고려해야 할 핵심 사항 (High Availability)

* pfSense의 위치: 현재 구조에서 Proxmox #1은 '머리' 역할을 합니다. 만약 하드웨어 가용 자원이 충분하다면, Proxmox #2에도 pfSense를 하나 더 설치하여 pfSense HA(High Availability)를 구성하는 것이 좋습니다. 그래야 #1 서버가 고장 나도 전체 망이 죽지 않습니다.
* 스토리지: 클러스터 구성 후 VM을 자유롭게 이동(Live Migration)시키려면 NAS(NFS/iSCSI)나 Ceph 같은 공유 스토리지를 스위치에 연결하는 것을 추천합니다.
```
