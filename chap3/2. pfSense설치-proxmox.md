# pfSense 설치

pfSense는 FreeBSD 기반이므로 설정 시 OS 타입을 정확히 지정하는 것이 중요하며, 방화벽 특성상 최소 2개의 네트워크 어댑터(WAN, LAN)가 필요합니다. 단계별 가이드는 다음과 같습니다.

---

## 1. 사전 준비

* **ISO 다운로드:** [pfSense 공식 홈페이지](https://www.pfsense.org/download/)에서 `Architecture: AMD64 (64-bit)`, `Installer Type: DVD Image (ISO) Installer`를 선택하여 다운로드 후 압축을 풉니다.

* Proxmox 관리 웹 화면에서 왼쪽 Node를 클릭하여 local -> ISO 이미지를 선택한 다음 pfSense ISO 이미지를 업로드하여 추가합니다.

## 2. 가상 머신(VM) 생성

1. VM 생성 버튼을 클릭합니다.

9. 확인 Tab에서 종결 버튼을 클릭하여 가상 머신을 생성합니다.

## pfSense 설정

1. pfSense 재부팅이 완료되고 나면 즉시 설정에 관련된 질문이 시작됩니다.
VLAN 설정을 지금 할지를 묻습니다. 지금 하지 않아도 나중에 할 수 있습니다. y|n 중에 n 을 입력합니다.

![alt text](image-16.png)

2. pfSense 가상 시스템을 구성할 때 2개의 네트워크 어댑터를 할당하고, pfSense 에서 2개의 네트워크 인터페이스를 발견했고, 각각 vtnet0 과 vtnet1 인데, 어떤것이 WAN 에 해당되고, LAN 에 해당되는지를 확인해서 결정하라는 것입니다.
 WAN의 인터페이스 이름으로 vtnet0을 입력하고 [Enter]를 클릭합니다.

![alt text](image-17.png)

3. 이번에는 LAN 인터페이스 이름으로 vtnet1을 입력하고 [Enter]를 클릭합니다.

![alt text](image-18.png)

4. WAN -> vtnet0, LAN -> vtnet1 설정했는데 맞는지 확인을 하라고 하는 것입니다. y|n 중에 y 를 입력합니다.

![alt text](image-19.png)

5. 마무리 설정 작업을 진행합니다.

![alt text](image-20.png)

6. 설치가 종료되고, WAN 인터페이스는 DHCP에 의해서 192.168.3.102 IP 주소가 할당되었고, LAN 인터페이스는 192.168.1.1 이 할당된 것이 출력됩니다

![alt text](image-21.png)

7. WAN, LAN IP 주소값을 모두 변경합니다.
WAN은 외부에서 접근해야 하는 인터페이스이기 때문에 DHCP 보다는 Static 으로 고정시키는게 좋습니다. LAN 은 내부 VM 들에 대한 네트워크 영역대이므로 내가 사용하기 쉬운 영역을 설정하는 것이 좋습니다.

각 인터페이스에 대해서 IP 주소를 할당하는 작업은 2) Set interface(s) IP address 메뉴를 선택해서 변경하시면 됩니다

![alt text](image-22.png)

8. WAN 인터페이스에 대해서 IP 주소를 설정하기 위해 1번을 선택하고 [Enter]를 클릭합니다.

![alt text](image-23.png)

9. IP 주소 설정을 DHCP로 하려면 y, Static 으로 하려면 n을 입력하고 [Enter]를 클릭합니다.

![alt text](image-24.png)

10. WAN의 IP 주소는 192.168.3.3으로 입력을 합니다.

![alt text](image-25.png)

## pfSense 웹 접속하기

아래의 모든 작업은 Proxmox 웹 관리 화면, 콘솔에서 진행합니다.

pfSense는 기본적으로:
- LAN → 접근 허용
- WAN → 접근 차단 (웹, ping 포함)

즉,WAN IP로 접속하려면 방화벽 정책을 직접 열어야 함

1. 외부 WAN IP로 접근을 하려고 합니다. WAN IP로 접근을 하려고 하면 임시로 방화벽을 비활성해야 합니다.

pfSense 콘솔에서:

```bash
8) Shell
```

PF 비활성화 (임시)

```bash
pfctl -d
```

![alt text](image-36.png)

>다음에 웹 관리 화면에 pfSense의 상태를 변경하면 자동으로 활성화 됩니다.
그래서 설정을 완료 할 때까지 계속 PF 방화벽을 비활성화 해야합니다.

2. 관리용 노트북에서 브라우저 실행하고 주소창에 WAN IP 192.168.3.3 입력합니다. 
기본 계정 : admin/pfsense

8. WAN 인터페이스 설정

메뉴 이동:
  Interfaces -> WAN (vtnet0)

Reserved Networks 부분
  - Block private networks and loopback addresses : 선택 해제

  RFC 1918(10/8, 172.16/12, 192.168/16)에 따라 개인 네트워크용으로 예약된 IP 주소, RFC 4193에 따른 고유 로컬 주소(fc00::/7) 및 루프백 주소(127/8)의 트래픽을 차단 합니다. 이 네트워크 인터페이스가 개인 주소 공간에 상주하지 않는 한 일반적으로 이 옵션을 켜야 합니다.

>그러나 우리의 WAN 선이 공용 아이피가 아니기 때문에 우리는 선택을 해제합니다.

  - Block bogon networks : 선택 해제

예약된 IP 주소(RFC 1918 제외) 또는 IANA에서 아직 할당하지 않은 주소의 트래픽을 차단합니다. Bogon은 인터넷 라우팅 테이블에 절대 표시되어서는 안 되는 접두사이므로 수신된 패킷의 소스 주소로 표시되어서는 안 됩니다.
이 옵션은 외부 인터페이스(WAN)에서만 사용해야 하며 로컬 인터페이스에서는 필요하지 않으며 잠재적으로 필요한 로컬 트래픽을 차단할 수 있습니다.
참고: 업데이트 빈도는 시스템 > 고급, 방화벽 및 NAT 설정에서 변경할 수 있습니다.

>그러나 우리의 WAN 선이 공용 아이피가 아니기 때문에 우리는 선택을 해제합니다.
