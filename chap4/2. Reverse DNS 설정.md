# 목표 (Reverse DNS 결과)

| IP          | PTR 결과         |
| ----------- | ---------------- |
| 192.168.0.5 | ns.kosa.co.kr    |
| 192.168.0.6 | team1.kosa.co.kr |
| 192.168.0.7 | team2.kosa.co.kr |

👉 `dig -x 192.168.0.6` 실행 시
👉 **team1.kosa.co.kr 반환**

---

# 1️⃣ Reverse Zone 개념 정리 (중요)

* 네트워크: **192.168.0.0/24**
* Reverse Zone 이름:

```
0.168.192.in-addr.arpa
```

📌 IP를 **거꾸로** 쓰고 마지막 옥텟은 레코드로 분리

---

# 2️⃣ Reverse Zone 선언

### 📄 `/etc/bind/named.conf.local`

기존 내용 아래에 **추가**

```conf
zone "0.168.192.in-addr.arpa" {
    type master;
    file "/etc/bind/db.192.168.0";
};
```

---

# 3️⃣ Reverse Zone 파일 생성

### 📄 `/etc/bind/db.192.168.0`

```bash
sudo nano /etc/bind/db.192.168.0
```

```dns
$TTL 86400
@   IN  SOA ns.kosa.co.kr. admin.kosa.co.kr. (
        2025121901  ; Serial
        3600        ; Refresh
        1800        ; Retry
        604800      ; Expire
        86400       ; Minimum
)

; Name Server
@       IN  NS  ns.kosa.co.kr.

; PTR Records
5       IN  PTR ns.kosa.co.kr.
6       IN  PTR team1.kosa.co.kr.
7       IN  PTR team2.kosa.co.kr.
```

📌 **주의**

* PTR 대상은 **FQDN + 점(.)**
* `5 = 192.168.0.5`

---

# 4️⃣ 문법 검사 (필수)

```bash
sudo named-checkconf
```

```bash
sudo named-checkzone 0.168.192.in-addr.arpa /etc/bind/db.192.168.0
```

✔ `OK` 나오면 정상

---

# 5️⃣ BIND 재시작

```bash
sudo systemctl restart bind9
```

---

# 6️⃣ Reverse DNS 테스트

### DNS 서버 또는 클라이언트에서

```bash
dig -x 192.168.0.6
```

정상 결과 예시

```
6.0.168.192.in-addr.arpa. 86400 IN PTR team1.kosa.co.kr.
```

또는

```bash
nslookup 192.168.0.7
```

```
7.0.168.192.in-addr.arpa name = team2.kosa.co.kr.
```

---

# 7️⃣ Forward ↔ Reverse 일치 확인 (중요)

```bash
dig team1.kosa.co.kr
dig -x 192.168.0.6
```

👉 **서로 일치해야 정상 구성**

# 🚨 실무 체크 포인트

| 항목                 | 체크        |
| ------------------ | --------- |
| PTR 대상 FQDN        | 끝에 `.` 있음 |
| Reverse Zone 이름    | IP 반전 정확  |
| Serial 증가          | 변경 시 필수   |
| Forward/Reverse 일치 | 매우 중요     |
