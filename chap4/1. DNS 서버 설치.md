# 3️⃣ BIND 기본 옵션 설정

### 📄 `/etc/bind/named.conf.options`

```bash
sudo vi /etc/bind/named.conf.options
```

```conf
options {

    // DNS 캐시 및 임시 데이터 저장 디렉토리
    // 외부 DNS 조회 결과를 캐싱하여 성능 향상
    directory "/var/cache/bind";

    // 재귀 쿼리 허용 여부
    // yes: 클라이언트 대신 외부 DNS까지 조회 수행 (일반적인 내부 DNS 서버 필수)
    recursion yes;

    // 재귀 요청을 허용할 클라이언트 범위
    // 내부망(192.168.0.0/24)만 허용 → 오픈 리졸버 방지 (보안 핵심)
    allow-recursion {
        localhost;
        192.168.0.0/24;
    };

    // DNS 서버가 요청을 수신할 IP 주소
    // localhost와 내부 IP만 허용 → 외부(WAN) 접근 차단 효과
    listen-on {
        127.0.0.1;        // 로컬 프로세스용
        192.168.0.5;      // 내부 사용자용 DNS IP
    };

    // DNS 질의 자체를 허용할 클라이언트 범위
    // allow-recursion과 별개로 "질문 자체"를 제한
    // 내부망만 허용 → 외부에서 DNS 사용 불가
    allow-query {
        localhost;
        192.168.0.0/24;
    };

    // 포워딩할 상위 DNS 서버 설정
    // BIND가 직접 루트부터 조회하지 않고 아래 DNS로 요청 전달
    // Google Public DNS 사용
    forwarders {
        8.8.8.8;   // Primary
        8.8.4.4;   // Secondary
    };

    // DNSSEC 검증 활성화
    // auto: 루트 trust anchor 자동 관리 (RFC 5011)
    // DNS 위변조 방지 (보안 기능)
    dnssec-validation auto;

    // IPv6 DNS 서비스 비활성화
    // IPv6 환경이 아니라면 불필요 → 공격 표면 감소
    listen-on-v6 { none; };
};
```

📌 의미

* **내부망(192.168.0.0/24)만 DNS 질의 허용**
* 외부 DNS(구글)로 포워딩 가능

# 4️⃣ Zone 선언 파일 설정

### 📄 `/etc/bind/named.conf.local`

```bash
sudo vi /etc/bind/named.conf.local
```

```conf
zone "kosa.co.kr" {
    type master;
    file "/etc/bind/db.kosa.co.kr";
};
```

# 5️⃣ 정방향 Zone 파일 생성

### 📄 `/etc/bind/db.kosa.co.kr`

```bash
sudo vi /etc/bind/db.kosa.co.kr
```

```dns
$TTL 3600
; 기본 TTL(Time To Live) 값
; DNS 레코드가 캐시에 유지되는 시간 (초 단위)
; 3600초 = 1시간
; → 한번 조회된 DNS 정보는 최대 1시간 동안 캐싱됨

; IN : Internet 클래스
; SOA (Start Of Authority) : 이 DNS Zone의 권한 시작(Authority) 정보, 이 도메인을 관리하는 최상위 정보
@   IN  SOA ns.kosa.co.kr. admin.kosa.co.kr. (
        2025121901  ; Serial
                    ; 존 파일 버전 번호 (매우 중요)
                    ; 형식: YYYYMMDDnn (권장)
                    ; 값이 변경되면 Secondary DNS가 변경 감지 후 zone transfer 수행

        3600        ; Refresh (1시간)
                    ; Secondary DNS가 Primary DNS에 변경 여부를 확인하는 주기

        1800        ; Retry (30분)
                    ; Refresh 실패 시 재시도 간격

        604800      ; Expire (7일)
                    ; Secondary DNS가 Primary와 통신 불가 시
                    ; 이 시간이 지나면 zone 데이터 폐기

        86400       ; Minimum (1일)
                    ; 과거: negative cache TTL (존재하지 않는 도메인 캐싱 시간)
                    ; 현재: 일부 resolver에서만 사용됨
)

; -------------------------------
; Name Server 설정
; -------------------------------

@       IN  NS  ns.kosa.co.kr.
; 이 도메인(kosa.co.kr)의 공식 네임서버 지정
; @ 는 kosa.co.kr 자체를 의미
; 즉, "kosa.co.kr의 NS는 ns.kosa.co.kr이다"

; -------------------------------
; DNS 서버 IP 정의
; -------------------------------

ns      IN  A   192.168.0.5
; ns.kosa.co.kr → 192.168.0.5
; 위 NS 레코드에서 지정한 네임서버의 실제 IP 주소
; 반드시 필요 (없으면 DNS 동작 안함)

; -------------------------------
; kosa.co.kr DNS
; -------------------------------
@       IN  A   192.168.0.5

; -------------------------------
; Team1 서비스
; -------------------------------

team1           IN  A   192.168.0.6
; team1.kosa.co.kr → 192.168.0.6

www.team1       IN  A   192.168.0.6
; www.team1.kosa.co.kr → 192.168.0.6
; 보통 웹서비스용 서브도메인

; -------------------------------
; Team2 서비스
; -------------------------------

team2           IN  A   192.168.0.7
; team2.kosa.co.kr → 192.168.0.7

www.team2       IN  A   192.168.0.7
; www.team2.kosa.co.kr → 192.168.0.7

```

📌 핵심 포인트

* `team1.kosa.co.kr` → **192.168.0.6**
* `www.team1.kosa.co.kr` → **192.168.0.6**

# 9️⃣ 클라이언트 DNS 설정

### 📌 192.168.0.10 ~ 192.168.0.100 PC에서

DNS 서버를 **192.168.0.5** 로 설정

#### Ubuntu 예시

```bash
sudo nano /etc/netplan/00-installer-config.yaml
```

```yaml
nameservers:
  addresses:
    - 192.168.0.5
```

```bash
sudo netplan apply
```

#### Windows

```
네트워크 설정 → IPv4 → DNS 서버
기본 DNS: 192.168.0.5
```
