# 1️⃣ Reverse Zone 개념 정리 (중요)

* 네트워크: **192.168.80.0/24**
* Reverse Zone 이름:

```
80.168.192.in-addr.arpa
```

📌 IP를 **거꾸로** 쓰고 마지막 옥텟은 레코드로 분리

---

# 2️⃣ Reverse Zone 선언

### 📄 `/etc/bind/named.conf.local`

기존 내용 아래에 **추가**

```conf
zone "80.168.192.in-addr.arpa" {
    type master;
    file "/etc/bind/db.192.168.80";
};
```

---

# 3️⃣ Reverse Zone 파일 생성

### 📄 `/etc/bind/db.192.168.80`

```bash
sudo vi /etc/bind/db.192.168.80
```

```dns
$TTL 86400
; 기본 TTL(Time To Live)
; 모든 레코드의 캐시 유지 시간 (초 단위)
; 86400초 = 24시간 → 클라이언트/캐시 DNS 서버가 하루 동안 결과를 유지

@   IN  SOA ns.kosa.co.kr. admin.kosa.co.kr. (
; SOA(Start Of Authority) 레코드
; 해당 Zone의 "관리 정보"를 정의하는 가장 중요한 레코드

        2025121901  ; Serial
; Zone 파일 버전 번호 (매우 중요)
; 변경 시 반드시 증가해야 DNS 서버가 변경사항을 인식
; 보통 YYYYMMDDNN 형식 사용

        3600        ; Refresh
; Secondary DNS가 Primary DNS에 변경 여부를 확인하는 주기 (초)
; 3600초 = 1시간마다 동기화 확인

        1800        ; Retry
; Refresh 실패 시 재시도 간격
; 1800초 = 30분 후 다시 시도

        604800      ; Expire
; Secondary DNS가 Primary와 통신 불가 시 유지 가능한 최대 시간
; 7일 이후에는 해당 Zone을 무효로 간주

        86400       ; Minimum
; 최소 TTL (예전에는 Negative Cache TTL로 사용)
; 존재하지 않는 도메인 응답을 캐싱하는 시간
)

; Name Server
@       IN  NS  ns.kosa.co.kr.
; 이 Zone을 관리하는 네임서버 지정
; @ = 현재 Zone (80.168.192.in-addr.arpa)
; 즉, "이 Reverse Zone은 ns.kosa.co.kr이 관리한다"

; PTR Records
5       IN  PTR ns.kosa.co.kr.
; 192.168.80.5 → ns.kosa.co.kr
; 실제로는 "5.80.168.192.in-addr.arpa" 를 의미
; DNS 서버 자신의 Reverse 매핑

6       IN  PTR team1.kosa.co.kr.
; 192.168.80.6 → team1.kosa.co.kr
; Reverse DNS 핵심 레코드
; dig -x 192.168.80.6 실행 시 반환됨

7       IN  PTR team2.kosa.co.kr.
; 192.168.80.7 → team2.kosa.co.kr
; team2 서버 Reverse 매핑
```

📌 **주의**

* PTR 대상은 **FQDN + 점(.)**
* `5 = 192.168.80.5`

---

# 4️⃣ 문법 검사 (필수)

```bash
sudo named-checkconf
```

```bash
sudo named-checkzone 80.168.192.in-addr.arpa /etc/bind/db.192.168.80
```

✔ `OK` 나오면 정상

---

# 5️⃣ BIND 재시작

```bash
sudo systemctl restart bind9
```

# 6️⃣ Reverse DNS 테스트

### DNS 서버 또는 클라이언트에서

```bash
dig @192.168.80.5 -x 192.168.80.6
```

정상 결과 예시

```
6.80.168.192.in-addr.arpa. 86400 IN PTR team1.kosa.co.kr.
```

또는

```bash
nslookup 
> server 192.168.80.5 (입력해하고 enter를 눌러주세요)
Default server: 192.168.80.5
Address: 192.168.80.5#53
> 192.168.80.7        (입력해하고 enter를 눌러주세요)
7.80.168.192.in-addr.arpa       name = team2.kosa.co.kr.
```

# 7️⃣ Forward ↔ Reverse 일치 확인 (중요)

```bash
dig team1.kosa.co.kr
dig -x 192.168.80.6
```

👉 **서로 일치해야 정상 구성**
