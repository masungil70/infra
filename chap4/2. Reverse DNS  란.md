
# Reverse DNS 정의

> **IP 주소 → 도메인 이름으로 변환하는 DNS (PTR 레코드)**

```
192.168.80.6  →  team1.kosa.co.kr
```

---

# Reverse DNS 사용 상황 5가지

---

## 1. 메일 서버

### 📌 상황

메일 서버가 이메일을 보낼 때

### 🔍 동작 과정

1. 상대방 메일 서버가 IP 확인
2. Reverse DNS 조회 (PTR)
3. Forward DNS와 일치하는지 확인

### ❗ 문제 상황

* PTR 없음 → 스팸 처리
* PTR ≠ A 레코드 → 거부

### 예시

```
IP: 203.0.113.10
PTR: mail.kosa.co.kr
A: mail.kosa.co.kr → 203.0.113.10
```

👉 **Forward ↔ Reverse 일치 필수**

---

## ✅ 2. 보안 로그 / 접근 제어

### 📌 상황

서버 로그 분석

### 예시

```
192.168.80.6 → team1.kosa.co.kr
```

👉 IP 대신 **이름으로 식별 가능**

### 활용

* SSH 접속 로그
* 웹 서버 접근 로그
* 방화벽 로그

---

## ✅ 3. 방화벽 / IDS / IPS 정책

### 📌 상황

특정 호스트 기반 제어

### 예시

* `team1.kosa.co.kr` 허용
* 특정 서버만 DB 접근

👉 내부망에서 특히 유용

---

## ✅ 4. 서비스 신뢰성 검증 (보안)

### 📌 상황

서버 간 통신 시 상대방 검증

예:

* API 서버
* 금융 시스템
* 내부 인증 시스템

👉 Reverse DNS로 “신뢰 가능한 호스트인지” 확인

---

## ✅ 5. 네트워크 진단 / 트러블슈팅

### 📌 상황

문제 분석 시

```bash
traceroute
dig -x
nslookup
```

👉 IP → 이름 변환으로
**가독성 + 분석 속도 향상**

---

# 4️⃣ Forward DNS vs Reverse DNS 관계

| 항목  | Forward | Reverse |
| --- | ------- | ------- |
| 방향  | 이름 → IP | IP → 이름 |
| 레코드 | A       | PTR     |
| 목적  | 접속      | 검증/식별   |

---

## 🔥 실무 핵심 규칙

👉 반드시 일치해야 함

```
team1.kosa.co.kr → 192.168.80.6
192.168.80.6 → team1.kosa.co.kr
```

👉 이걸 **Forward-Confirmed Reverse DNS (FCrDNS)** 라고 함
